# Leading Questions — Socratic Self-Interrogation Before Design

> These questions are asked by the harness to ITSELF before any visual work begins. They force the designing agent out of BFS ("everything at 85%") into DFS ("nail one thing perfectly"). Based on axieomatic's feedback: "The question you need to make the agent ask itself is: what would a highly opinionated designer build to make THIS specific ICP resonate deeply with this vision?"

## The Six Questions

### 1. Opinionated Designer Test

"What would a highly opinionated designer build to make THIS specific ICP resonate deeply with this vision?"

**Why this matters:** Generic agents produce generic designs. The agent must adopt a POINT OF VIEW — a specific aesthetic stance informed by the ICP, not a safe middle-ground. If you can't articulate your design opinion in one sentence, you don't have one yet.

### 2. Income Calibration

"Is this design for someone who makes $50K or $500K? How does that change the visual language?"

**Why this matters:** A freelancer making $80K and a CFO making $400K respond to fundamentally different visual cues. Luxury signals (whitespace, serif fonts, muted palettes) vs. value signals (social proof, pricing clarity, feature density) are not interchangeable. Get the income bracket wrong and the entire design speaks the wrong language.

### 3. Eight-Second Clarity

"If this user has 8 seconds on the page, what ONE thing must they understand?"

**Why this matters:** Forces hierarchy. If you can't name the single takeaway, the page has no focal point. Everything above the fold must serve this one message. Every other element is subordinate. If you have two "most important things," you have zero.

### 4. Recognition Test

"What would make this user say 'finally, someone gets it' vs 'oh, another AI tool'?"

**Why this matters:** The difference between a design that converts and one that bounces is RECOGNITION. The user must see their own problem reflected back at them in the first viewport. Use their language, their context, their frustration. Not your marketing copy.

### 5. Minimum Viable Page

"What is the LEAST I can put on this page and still convert?"

**Why this matters:** Every element you add dilutes the ones already there. The BFS trap is adding more sections, more features, more proof points. The DFS approach: find the minimum set of elements that produce conversion, then make each one exceptional. Subtract before you add.

### 6. Vertical Signal Test

"If I showed this to the ICP without any text, would they know what vertical this is for?"

**Why this matters:** Visual language encodes domain. A fintech page should FEEL like finance before you read a word (structured grids, muted confidence, data density). A creative tool should FEEL creative (expressive layout, bold color, dynamic elements). If your design could belong to any vertical, it belongs to none.

## How to Use These Questions

1. Answer ALL six BEFORE opening any design tool or writing any CSS
2. Write answers in the discovery brief under a "## Socratic Answers" section
3. If any answer is "I don't know" — STOP. Research until you do.
4. Reference these answers during design reviews to check alignment
5. If the design doesn't match the answers, the design is wrong — not the answers
